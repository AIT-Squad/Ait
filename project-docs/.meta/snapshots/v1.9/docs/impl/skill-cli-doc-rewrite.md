## init 阶段注入 skill 路径并生成项目本地 wrapper

<!-- @ref:prd-skill-cli-resolution rel:implements -->

### 实现要点

#### 1. wrapper 向下传递自身路径
- `skill/ait/bin/ait`：在 `exec` 之前 `export AIT_SKILL_DIR="$SKILL_DIR"`，把已解析好的 skill 安装路径透传给 Python 进程。
- `skill/ait/bin/ait.cmd`：对应增加 `set "AIT_SKILL_DIR=%SKILL_DIR%"` 后再调用 `%VENV_PY%`。
- 不读 `cwd`、不读 `argv[0]`，CLI 端只信任 `os.environ["AIT_SKILL_DIR"]`；缺失则按 `~/.claude/skills/ait` 兜底。

#### 2. CLI 在 init 时落盘 skill_dir / cli_path
改造点：`skill/ait/ait/init_manager.py` 的 `InitManager._mark_initialized`。
- 写入字段（追加，不覆盖既有键）：
  - `skill_dir`: `os.environ.get("AIT_SKILL_DIR", str(Path.home() / ".claude" / "skills" / "ait"))`
  - `cli_path`: `<skill_dir>/bin/ait`（Windows 下为 `<skill_dir>/bin/ait.cmd`，按 `os.name == "nt"` 分支）
  - `wrapper_path`: `project-docs/.ait/ait-cli`（项目本地相对路径，文档统一引用此值）
- 已有 `initialized: true` 行为保持不变；新增字段独立于 `initialized` 写入，避免回填阻塞。

#### 3. 生成项目本地薄壳 `project-docs/.ait/ait-cli`
- 在 `InitManager.run` 末尾、`_mark_initialized` 之后新增 `_write_project_wrapper` 步骤。
- 目录结构：
  - 类 Unix：`<root>/.ait/ait-cli`（`chmod 0755`）
  - Windows：`<root>/.ait/ait-cli.cmd`
- 内容（Unix）：
  ```bash
  #!/usr/bin/env bash
  # AIT project-local wrapper — generated by `ait init`. Do not edit manually.
  exec "{cli_path}" "$@"
  ```
- 内容（Windows）：
  ```bat
  @echo off
  "{cli_path}" %*
  ```
- `{cli_path}` 由步骤 2 的 `cli_path` 字段渲染。
- 同时生成 `<root>/.ait/.gitignore`（内容 `*` + `!.gitignore` + `!ait-cli` + `!ait-cli.cmd`），可选；本期默认不写，让 `.ait/` 随项目入库以保证团队成员路径一致——当 skill 安装路径不同时，由 `init --refresh-wrapper` 重写。

#### 4. 新增 `bin/ait init --refresh-wrapper`
- 仅重写 `.meta/config.yaml` 的 `skill_dir`/`cli_path`/`wrapper_path` 三个字段与 `project-docs/.ait/ait-cli*` 文件，**不改任何 docs/global/\***。
- 适用场景：用户换机 / skill 安装路径变化 / 团队成员从 git 拉取后第一次本地化。
- 输出：`{"ok": true, "data": {"refreshed": true, "skill_dir": "...", "wrapper": "project-docs/.ait/ait-cli"}}`。
- 与 `prd-init-idempotent` 引入的 `init --check` 共存，互不干扰。

#### 5. CLI 入口提示
`skill/ait/ait/cli.py` 在以下两种情况下追加 stderr 提示（不污染 stdout JSON）：
- 检测到 `cwd` 含 `project-docs/` 但 `project-docs/.ait/ait-cli` 不存在 → 提示 `tip: run \`{AIT_SKILL_DIR}/bin/ait init --refresh-wrapper\` to generate project-local wrapper`。
- 检测到 `.meta/config.yaml` 中 `skill_dir` 与当前 `AIT_SKILL_DIR` 不一致 → 提示 `tip: skill_dir mismatch detected, run init --refresh-wrapper`。

### 测试要点
- 单元：`tests/test_init_manager.py` 覆盖 config 字段写入、wrapper 文件生成、refresh-wrapper 不动 docs。
- 集成：在 tmp dir 跑 `bin/ait init` → 断言 `project-docs/.ait/ait-cli` 可执行且 `--version` 输出与 skill 版本一致。
- 跨平台：mock `os.name == "nt"` 分支断言生成 `.cmd` 文件。

### 边界
- 不修改 wrapper 已有的"自定位"逻辑，仅追加 `export AIT_SKILL_DIR`。
- 不引入 `AIT_HOME` 等用户可配置环境变量（保持显式路径调用哲学）。
- 不写 PATH、不创建系统级符号链接。

- 在 5 个不同 cwd（`/tmp`、`$HOME`、`$SKILL_DIR`、`$SKILL_DIR/bin`、随机临时目录）下分别绝对路径调用 `$SKILL_DIR/bin/ait --version`。
- 断言每次输出均为 `ait, version <X.Y.Z>` 且 exit 0。
- 断言进程内 `AIT_SKILL_DIR` 环境变量等于 `$SKILL_DIR`（通过 `bin/ait` 增加一个隐藏诊断子命令 `__diag` 输出环境变量？或直接在 venv 用 `python -c` 读取，脚本里实现）。

### 集成入口
- `skill/ait/scripts/verify-all.sh` 追加调用 `verify-wrapper-self-locate.sh`。
- 失败则脚本以 exit 1 退出，CI/本地 verify 都能捕获回归。

### 边界
- 不实现 Windows 版 `verify-wrapper-self-locate.cmd`（团队主开发环境为 macOS/Linux；Windows 由用户实际运行验证）。
- 不引入 `pytest` 依赖（保持 verify scripts 轻量）。
